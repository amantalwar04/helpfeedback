Map Attributes

Map attributes specify whether topics referenced within the map should be included in various outputs or generated content. These attributes apply also to elements within a bookmap.
Name Description Available Values Required
print Specifies whether the topic should be included in print outputs, such as PDF. The default is yes. yes | no Only if you want to exclude the topic from print output
@processing-role Specifies whether the topic should be included as a normal topic, or is simply used as a resource for resolving keyrefs and conkeyrefs.
Placeholder for example
 normal | resource-only Only when the topic should be used as a resource only; for example, on key files.
@toc Specifies whether topics are included in a site map or table of contents. The default is yes.
In this example, the content in the reference topic was broken out for reuse purposes only. This distinction is not important to the user, so the topic is excluded from the TOC, making it appear as though it is part of the parent topic.
Placeholder for example
 yes | no Only if you want to exclude the topic from the table of contents
@chunk Specifies that all nested topics be combined together as a single topic when processed. For example, although three topics are specified in the following example, they will be output as one topic when the map is processed.
Placeholder for example
 to-content Only if you want multiple topics to be displayed together as one topic in an outline output
@copy-to Specifies a different file name for a particular instance of the topic in the map. The links and navigation associated with that instance will point to a copy of the topic with the file name you specified, ensuring that context is maintained when users navigate to and from the topic instance.
For example, suppose you have two installation scenarios, each of which require the same beginning and ending tasks. To ensure that users stay within their desired installation scenario as they click forward and back, each topic must have a different name within this specific map. To accomplish this, you reference the same file, but provide a temporary name each time you reference the file.
